webpack的基本使用

一月 04, 2022 4125

关于webpack

1、什么是webpack

​ webpack是一个前端构建工具，提供了模块化支持，代码压缩混淆，解决js兼容问题，性能优化等特性，能够提高开发效率和项目的可维护性。

2、如何使用webpack

​ 创建一个用于转换es6语法的小例子：
​

  1. 新建项目文件夹、使用npm init -y 初始化一个项目，创建src文件夹用于存储源代码文件，src文件夹下有index.js和index.html文件，在index.html中引用index.js中的含有es6语法的文件。直接打开index.js后会报错。
  2. 使用npm install webpack@4.43.3 webpack-cli@3.3.12安装webpack，安装完成后在项目文件夹根目录创建webpack.config.js文件用于配置webpack，并在package.json中配置dev。
  1. 配置完成后执行 npm run dev 就会在dist文件夹下创建一个打包好的main.js文件，在index.js中引入这个文件即可。

3、自定义webpack打包的入口和出口

webpack默认打包入口为src>index.js,默认出口为dist>main.js

4、设置自动打包

默认每次打包后都要重新使用npm run dev进行打包，可以使用webpack-dev-derver实现自动打包，即每次保存文件后系统会自动帮我们进行打包，配置方法如下：

  • 通过npm i webpack-dev-server下载webpack-dev-server

  • 在package.json中修改dev为”dev”: “webpack”

  • 在index.html中设置引用的src为bundle.js

  • 注意：webpack-dev-server自动打包的输出文件，默认放到了服务器的根目录中.

  • 实现运行后自动打开页面 “dev”: “webpack-dev-server –open –host 127.0.0.1 –port 9999”

5、配置html-webpack-plugin

html-webpack-plugin能够实现设置默认打开的页面，设置方式如下：

  • 下载 npm i html-webpack-plugin
  • 在webpack.config.js中进行配置

    6、webpack中的加载器

默认情况下，webpack职能打包js类型的文件，想要打包非js类型的文件需要使用loader
以css样式文件的处理为例进行介绍：

  • 首先需要下载style-loader、css-loader来处理样式文件 npm install style-loader css-loader
  • 在webpack.config.js中进行配置

高级js的打包：打包js文件中的高级语法：在编写js的时候，有时候我们会使用高版本的js语法
有可能这些高版本的语法不被兼容，我们需要将之打包为兼容性的js代码
我们需要安装babel系列的包
